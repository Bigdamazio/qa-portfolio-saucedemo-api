# Plano de Testes

## 1. Objetivo

Validar os principais fluxos do sistema SauceDemo e complementar o portfólio com testes de API pública, demonstrando organização, pensamento crítico e documentação de QA.

## 2. Escopo

### Dentro do escopo
- Login com credenciais válidas
- Login com credenciais inválidas
- Login com campos vazios
- Adicionar produto ao carrinho
- Remover produto do carrinho
- Finalizar compra
- Validação de mensagens e comportamento de interface
- Testes de API com cenários positivos e negativos

### Fora do escopo
- Testes de performance
- Testes de carga
- Testes de segurança ofensiva
- Automação completa neste primeiro momento

## 3. Ambiente de teste

### Web
- Aplicação: SauceDemo
- Navegador principal: Google Chrome
- Apoio: DevTools

### API
- Ferramenta: Postman
- API sugerida: DummyJSON ou Reqres

## 4. Critérios de entrada
- Aplicação disponível
- Internet funcionando
- Ferramentas instaladas
- Conta ou credenciais de teste disponíveis

## 5. Critérios de saída
- Casos principais executados
- Evidências registradas
- Bugs documentados
- Resultado consolidado no repositório

## 6. Riscos
- Ambiente demo indisponível
- Instabilidade de API pública
- Bugs não reproduzíveis
- Mudanças no comportamento do sistema demo

## 7. Entregáveis
- Plano de testes
- Casos de teste
- Relatório de bugs
- Checklist de validação
- Evidências
